## Назначение

Поле редактора позволяет вводить данные в формате WYSIWYG для некоторого контента, связанного со статьей, в дополнение к основному контенту.


## Создание поля

Специальные параметры в этом поле:

- **Кнопки** Показать или скрыть выпадающий список CMS Content. В случае с дополнительным текстом статьи может быть нецелесообразно показывать некоторые или все кнопки в списке. Значение по умолчанию для плагина — Скрыть.
- **Скрыть кнопки** Если параметр *Кнопки* установлен на *Да* или значение по умолчанию для плагина — *Показать*, скрыть выбранные кнопки в выпадающем списке CMS Content.
- **Ширина и высота** значения включают пространство, занимаемое панелью инструментов редактора, поэтому лучше оставить их пустыми на начальном этапе.
- **Ширина** Значение ширины определяет ширину WYSIWYG редактора в % или пикселях. Значение по умолчанию — 100%.
- **Высота** Значение высоты определяет высоту (в пикселях) WYSIWYG редактора. Значение по умолчанию для этого — 250px. Значение может быть представлено как дробь от высоты окна просмотра, например 50vh.
- **Фильтр** Позволяет системе сохранять определенные html-теги или исходные данные.

## Ввод данных

В форме редактирования статьи дополнительное поле редактора похоже на основное поле редактора контента.

## Отображение данных

Следующий скриншот сайта показывает поле, отображаемое в статье. Опция *Автоматическое отображение* отвечает за позицию поля, а ваш шаблон отвечает за дизайн поля.

В отображении статьи введенный текст появляется под заголовком, но является частью маркированного списка для этого элемента поля.

Ищите элемент **Заметки по культивации**.
